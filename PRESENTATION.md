# [발표 스크립트] 나만의 식도락 지도 (Gourmet Map) 발표 및 시연

이 발표용 스크립트는 본 서비스의 기획 의도부터 핵심 기능 시연 시나리오, 기술 스택, 그리고 향후 계획까지 자연스럽고 설득력 있는 구어체 발표 멘트로 작성되었습니다.

---

## 1. 도입부 & 문제 정의 (Slide 1)

**[멘트]**
> 안녕하세요. 오늘 제가 소개할 프로젝트는 프라이빗 맛집 관리 솔루션, **'나만의 식도락 지도(Gourmet Map)'**입니다. 
> 
> 여러분은 오늘 점심에 갈 맛집을 고를 때 어떤 서비스를 사용하시나요? 보통 대다수의 맛집 서비스는 대중들의 별점과 리뷰, 그리고 수많은 마케팅 광고로 가득 차 있습니다. 광고성 블로그 글과 복잡한 화면 피로도 때문에, 정작 **"내가 진짜 인정하는 맛집"**이나 **"나중에 꼭 가봐야지 하고 메모해둔 장소"**를 깔끔하게 따로 관리하기란 무척 어렵습니다.
> 
> 저희는 바로 이 지점에서 문제를 정의했습니다. 대중의 시선이 아닌, **'오직 내 기록과 나의 경험만으로 채워나가는 직관적이고 직진성 있는 프라이빗 미식 지도'**를 만들어보자는 것이었습니다.

---

## 2. 핵심 기능 시연 (Slide 2 - 직접 화면을 조작하며 발표)

**[화면: 로그인 / 회원가입 창 띄움]**
**[멘트]**
> 실제 작동 화면을 시연하며 기능을 소개해 드리겠습니다. 
> 저희 서비스는 불필요한 이메일 인증 절차를 걷어낸 **간소화된 회원가입**을 지원합니다. 임의의 이메일과 비밀번호 입력만으로 즉시 계정이 생성되고 로그인이 완료됩니다.

**[행동: 로그인 버튼 클릭 -> 서울역 중심의 기본 지도가 펼쳐짐]**
**[멘트]**
> 로그인이 되면, 처음 들어온 사용자도 막막하지 않도록 **서울역을 중심으로 엄선된 5곳의 기본 식도락 데이터가 데이터베이스로부터 자동으로 시딩(Seeding)**되어 지도를 채워줍니다.

**[행동: 왼쪽 상단 검색창에 '홍대입구역' 타이핑 후 엔터]**
**[멘트]**
> 이제 지도 왼쪽 상단에 위치한 **'주소 및 지명 검색창'**을 이용해 보겠습니다. `홍대입구역`을 검색하면 카카오의 정밀 주소 API가 실시간 연동되어 지도가 해당 위치로 부드럽고 정확하게 이동하며 **하늘색 안내 마커**를 띄워줍니다.

**[행동: 지도 상단 중앙의 '이 지역 실제 맛집 검색' 클릭 -> 회색 핀들이 쫙 뿌려짐]**
**[멘트]**
> 자, 이제 홍대입구역 주변의 실제 맛집들을 등록해 볼 텐데요. 상단 중앙의 **'이 지역 실제 맛집 검색'** 버튼을 클릭하면, 카카오의 실시간 POI 엔진을 통해 **현재 화면 반경 1.5km 이내의 진짜 음식점과 카페 90곳**을 긁어옵니다. 
> 보시는 것처럼 제가 이미 등록한 맛집들을 제외한 새로운 장소들만 **회색 핀 마커**로 지도에 촘촘하게 표시됩니다.

**[행동: 회색 마커 중 하나를 클릭 -> 말풍선 팝업 노출 -> '내 맛집으로 등록' 클릭]**
**[멘트]**
> 이 중 마음에 드는 식당의 회색 마커를 클릭하고 **'내 맛집으로 등록'**을 누르면, 상호명, 주소, 경위도 좌표와 카테고리가 폼에 1초 만에 자동으로 세팅됩니다. 

**[행동: 등록 모달에서 메모를 적고 '가보고 싶은 곳' 상태로 저장 버튼 클릭]**
**[멘트]**
> 이 상태로 저장을 누르면, 회색 마커가 즉시 저만의 빨간색 위시리스트 마커로 변경되며 클라우드 데이터베이스에 실시간 영구 저장됩니다.

**[행동: 등록된 마커 중 하나를 클릭 -> 상세 정보 뷰어 노출 -> 별점 클릭 / 메모 텍스트 수정]**
**[멘트]**
> 상세 보기 화면에서는 UX의 몰입감을 극대화하기 위해 저장 버튼을 없앴습니다. 별점을 바꾸거나 메모장을 수정하면, **인라인 실시간 감지 로직을 통해 텍스트 입력 즉시 데이터베이스에 자동 저장(Auto-save)되고 하단에 비차단식 토스트 알림**이 나타납니다.

**[행동: 상단 중앙의 '숨기기' ❌ 버튼 클릭 -> 회색 핀들과 검색 결과가 사라짐]**
**[멘트]**
> 탐색이 끝나고 다시 나만의 정돈된 맛집 지도만 깔끔하게 보고 싶다면, 상단의 **'숨기기'** 버튼을 클릭하여 주변 검색 결과를 즉시 정리할 수 있습니다.

## 3. 기술 스택 (Slide 3)

**[멘트]**
> 다음으로 이 서비스를 구축하기 위해 채택한 기술 스택입니다. 
> 
> *   **프론트엔드**는 모바일에서도 쾌적하게 구동되도록 가볍고 빠른 성능을 위해 별도의 무거운 프레임워크나 빌더 없이 **Pure HTML5, Vanilla CSS, Vanilla JS**를 채택했습니다. 
> *   지도의 인터랙션과 마커 렌더링은 오픈소스 지도 라이브러리인 **Leaflet.js**와 대용량 핀을 그룹화해 주는 **markercluster** 라이브러리를 활용했습니다.
> *   **백엔드 및 데이터베이스**는 서버리스 BaaS 솔루션인 **Supabase**를 활용했습니다. 이를 통해 별도의 서버 구축 없이 보안 규칙(RLS)이 적용된 PostgreSQL 데이터베이스 및 사용자 인증 시스템을 안전하고 실시간으로 연동했습니다.
> *   **위치 정보 및 데이터 검색**은 대한민국에서 가장 정확한 데이터를 가진 **Kakao Maps Places API**와 오픈소스 주소 변환 엔진인 **OpenStreetMap Nominatim**을 결합해 하이브리드로 주소와 상호를 정교하게 매핑했습니다.
> *   **배포 파이프라인**은 깃허브 저장소와 연동된 **Vercel**을 통해 개발부터 호스팅까지 자동 CI/CD 라인을 구축했습니다.

## 4. 발전 방향 및 다음 버전 (Slide 4)

**[멘트]**
> 마지막으로 앞으로 개발해 나갈 다음 버전의 로드맵입니다.
> 
> *   **첫째, 공유 링크 생성 기능**입니다. 나만의 테마별 지도를 카카오톡이나 SNS 링크를 통해 친구들에게 '읽기 전용'으로 즉시 공유할 수 있는 기능을 구축하겠습니다.
> *   **둘째, Supabase Storage 연동 및 미디어 업로드**입니다. 텍스트 기록을 넘어, 사용자가 매장에서 직접 촬영한 음식 사진과 영수증 사진을 업로드하여 더욱 입체적인 미식 다이어리를 만들 예정입니다.
> *   **셋째, PWA(Progressive Web App) 기술 적용**입니다. 사용자가 모바일 브라우저 접속 후 홈 화면에 바로 가기 앱으로 추가하여, 앱스토어 설치 없이도 마치 네이티브 앱처럼 동작하는 최적화된 사용성을 선사할 것입니다.
> 
> 이상으로 광고 없는 초개인화 미식 플랫폼, '나만의 식도락 지도' 발표를 마치겠습니다. 들어주셔서 감사합니다.
